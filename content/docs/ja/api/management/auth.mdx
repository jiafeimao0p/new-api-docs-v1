## 説明

バックエンド管理インターフェースは多段階の認証メカニズムを採用しており、一般的なものとして：**公開**、**ユーザー**、**管理者**、**Root** があります。

## 認証方法（いずれかを選択）

### セッション

ログインインターフェースを通じてセッションを取得します：

- `POST /api/user/login`

### アクセストークン（推奨）

リクエストヘッダーに含めます：

```text
Authorization: Bearer {token}
```

トークンは「個人設定 - セキュリティ設定 - システムアクセストークン」で生成できます。

## 必須リクエストヘッダー

一部のインターフェースでは、ユーザー識別子リクエストヘッダーの携帯が要求されます：

```text
New-Api-User: {user_id}
```

ここで `{user_id}` は現在のログインユーザーと一致する必要があります。

## 権限レベル

- **公開（Public）**：認証不要
- **ユーザー（User）**：ログインまたはアクセストークンが必要
- **管理者（Admin）**：管理者権限が必要
- **Root**：最高権限
